# 🏋️ AI Physiotherapy Coach

An AI-powered physiotherapy exercise quality assessment system that uses skeleton motion data and a Transformer-based deep learning model to automatically classify whether a patient performed a physiotherapy exercise **correctly** or with an **error** — and, in future phases, identify the *specific type* of error made.

## 🧠 Project Overview

This project builds an intelligent coaching assistant for physiotherapy rehabilitation. It analyses 3D skeleton motion sequences captured during exercises and learns to distinguish correct technique from common errors.

**Core Task:** Binary classification — `0 = Correct`, `1 = Error`

**Future Task (V2):** Multi-class classification — `Correct`, `Error1 (Amplitude)`, `Error2 (Speed)`, `Error3 (Posture)`, etc.

The system is designed to ultimately power a **mobile application** that gives patients real-time, personalised feedback during home rehabilitation sessions.

## 📂 Dataset

The project uses the **Keraal physiotherapy dataset** from ENSTB (Brest, France). It contains motion-capture recordings of patients performing 3 specific back-rehabilitation exercises.

### Exercises

| Code | Full Name | Correct Reps | Error Reps | Total |
|------|-----------|:---:|:---:|:---:|
| **CTK** | Cross Trunk Kneel ("Bird-Dog") | 45 | 135 | 180 |
| **ELK** | Erect Lordosis Kneeling | 45 | 135 | 180 |
| **RTK** | Right Trunk Kneeling | 45 | 135 | 180 |
| **Total** | | **135** | **405** | **540** |

> ⚠️ The dataset has a **1:3 class imbalance** (Correct:Error), which is handled via class weighting during training.

### Data Modalities

| Modality | Sensor | Format | Used In |
|----------|--------|--------|---------|
| **Vicon** | Optical Motion Capture (gold standard) | `.txt` — 17 joints × 7 features/frame | V1 (Phase 1 & 2) |
| **Kinect** | Microsoft Kinect v2 | `.txt` — 25 joints × 7 features/frame | V2 (Phase 3+) |
| **OpenPose** | Camera + AI Pose Estimation | Structured files | V2 (Phase 3+) |
| **XML Annotations** | Expert annotator labels | `.anvil` (XML) | V2 label extraction |
| Video | Standard cameras | `.mp4`, `.avi` | Not used directly |

Each skeleton frame contains **7 features per joint**: `X`, `Y`, `Z` coordinates + quaternion rotation `(qw, qx, qy, qz)`.

## 📓 Notebooks

### `01_data_download.ipynb`
Downloads the three dataset archives (`group1A`, `group2A`, `group3`) from the Keraal/ENSTB server via `wget`, extracts them to Google Drive, and verifies the directory structure.

### `02_Data_Exploration_and_Preprocessing.ipynb`
- Loads raw Vicon `.txt` files and reshapes them from `(frames, 119)` → `(frames, 17, 7)`
- Visualises a single skeleton frame in 3D and as an animated sequence
- Applies **root-joint centering** (subtracts hip position from all joints per frame)
- Applies **torso-length scaling** (divides by the hip-to-neck distance for scale invariance)
- Defines a reusable `preprocess_skeleton()` function
- Batch-processes all 540 Vicon files, extracts labels from filenames (`C` = Correct, else Error)
- Pads all variable-length sequences to `MAX_LENGTH = 500` frames
- Saves the final dataset as `vicon_processed_data.npz` — shape `(540, 500, 17, 7)`

### `03_V1_Model_Training.ipynb`
- Loads `vicon_processed_data.npz` and splits into **Train (70%) / Val (15%) / Test (15%)** using stratified splitting
- Computes per-class weights to address the 1:3 imbalance:
  - Class 0 (Correct): weight ≈ **2.01**
  - Class 1 (Error): weight ≈ **0.67**
- Defines and trains the **V1 Transformer model** (see architecture below)
- Uses `ModelCheckpoint` + `EarlyStopping (patience=10)` callbacks
- Evaluates on the held-out test set with accuracy, classification report, and confusion matrix

### `04_V2_Data_Processing_XML.ipynb`
- Parses `.anvil` (XML) annotation files from both `annotatorA` and `annotatorB` using `xml.etree.ElementTree`
- Extracts per-repetition labels from `Global evaluation` and `Global error` tracks
- Combines both annotators into `df_v2_master_labels`
- Removes low-information `Incorrect` labels and saves `df_v2_labels_CLEAN.csv`

### `05_V2_Sequence_Preparation.ipynb`
- Loads `df_v2_labels_CLEAN.csv` and resolves corresponding Kinect/OpenPose files per repetition
- Applies the same normalization pipeline (center + scale + pad) with `MAX_LENGTH=500`, `NUM_JOINTS=25`, `NUM_FEATURES=7` at 30 FPS
- Assembles final NumPy arrays ready for V2 model training

## 🤖 Model Architecture (V1)

A custom **Transformer Encoder** built with TensorFlow / Keras.

```
Input Shape: (500, 17, 7)
    │
    ▼
Reshape → (500, 119)          # Flatten joints × features per frame
    │
    ▼
PositionalEmbedding
    ├── Dense(256)             # Token embedding: 119 → 256-dim
    └── Embedding(500, 256)   # Learned positional encoding
    │
    ▼
TransformerEncoder × 6        # 6 stacked encoder blocks
    ├── MultiHeadAttention(num_heads=8, key_dim=256)
    ├── Add & LayerNorm
    ├── Dense(FF_DIM=256, relu)
    └── Add & LayerNorm
    │
    ▼
GlobalAveragePooling1D         # (500, 256) → (256,)
    │
    ▼
Dropout(0.5)
Dense(128, relu)
Dropout(0.5)
    │
    ▼
Dense(2, softmax)              # Output: [P(Correct), P(Error)]
```

| Hyperparameter | Value |
|---|---|
| Total Parameters | 2,566,530 |
| Embed Dimension | 256 |
| Transformer Heads | 8 |
| Encoder Blocks | 6 |
| Feed-Forward Dim | 256 |
| Optimizer | Adam (lr = 1e-4) |
| Loss | `sparse_categorical_crossentropy` |
| Batch Size | 16 |
| Max Epochs | 50 |
